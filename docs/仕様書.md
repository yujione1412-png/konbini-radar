## 2. アプリの目的と3つの中核機能

依頼者が最初に挙げた要件は次の3つ。すべて実装済み。

1. **現在地周辺のコンビニを探す**（探す範囲を設定できる）
2. **A地点→B地点のルートを設定し、そのルート沿いのコンビニを探す**
3. **探した内容の履歴を残し、再表示できる**

> ※ 3について、依頼者は途中で「履歴に残したいのは**検索履歴ではなくルート表示履歴**」と
> 明確化した。**検索の履歴は保存しない。ルートを表示したときだけ履歴に残す。**

## 3. 技術構成（すべて無料・カード登録不要）

**単一の `index.html` に HTML / CSS / JavaScript をすべて内包する静的Webアプリ。**
ビルド不要。Vercelに置くだけで動くこと。フレームワークは使わない。

| 用途 | 使うもの | 備考 |
|---|---|---|
| 地図表示 | **Leaflet 1.9.4**（CDN: cdnjs） | |
| 地図タイル | **OpenStreetMap** | `© OpenStreetMap contributors` の帰属表示必須 |
| コンビニ検索（主） | **YOLP Yahoo!ローカルサーチAPI** | Client IDのみ。カード不要。1日5万件無料 |
| コンビニ検索（副） | **Overpass API**（OpenStreetMap） | Client ID未設定時のフォールバック |
| ルート検索 | **OSRM (FOSSGIS公開サーバー)** | キー不要。徒歩/車の2プロファイル |
| 外部地図連携 | **Google Maps URLs** | キー不要。アプリを開くだけ |
| 履歴保存 | **localStorage** | 端末内のみ。サーバー送信なし |

### 3.1 なぜこの構成なのか（意思決定の経緯 ― 覆さないこと）

- **Google Maps Platform は採用しない。**
  データは最良だが、キー取得に**クレジットカード登録が必須**で、無料枠超過時に課金リスクがある。
  さらに規約上、取得データはGoogleの地図上に表示する必要があり、OSMの地図に出すのは規約違反。
  依頼者の「完全無料・カード不要」条件に反するため不採用。

- **コンビニ検索は OSM だけでは実用にならなかった。**
  当初 Overpass API（OSM）のみで実装したが、**日本のコンビニが大量に登録漏れしており、
  実用に耐えなかった**（依頼者の実地確認による）。ブランド名での検索追加なども試したが改善せず。
  そこで **YOLP（Yahoo!ローカルサーチ）に切り替えて解決**した。
  YOLPは全国の電話帳データが元で網羅性が高い。**日本国内ではYOLPを主データ源とすること。**

- **YOLPはJSONPで呼ぶ。**
  ブラウザから直接 `fetch` するとCORSで弾かれる。YOLPは公式に `callback` パラメータ（JSONP）に
  対応しているため、**サーバーを立てずに静的サイトから直接呼べる**。この方式を維持すること。

- **ルート検索は OSRM の公開サーバー（FOSSGIS）。**
  キー不要・無料。徒歩と車でエンドポイントが違う。混雑時は失敗しうるので、
  失敗時は日本語のエラーメッセージを出し、Google Mapへの導線を案内する。

### 3.2 Yahoo! Client ID

```javascript
const YAHOO_CLIENT_ID = 'dmVyPTIwMjUwNyZpZD1ZU2xVUGlzZzRTJmhhc2g9WkRjME16ZzFZalZrTjJFeE16WmxOQQ';
```

- **空文字にすると自動的にOSM(Overpass)にフォールバックする**設計にすること（動作確認用）。
- Client IDはページソースに露出する。個人利用のため許容済み（依頼者了承済み）。

### 4.3 UI要素の詳細

**ヘッダー**
- タイトル「コンビニを探す」、右に「履歴」ボタン、右端にバージョン表示（例 `v0.6.0`）。

**操作バー**
- **探す範囲スライダー**: 200m〜3000m、100m刻み、初期値500m。現在値をラベル表示。
- **「このあたりを探す」ボタン**: 周辺検索を実行。
- **出発トグル**（現在地 / 指定中心）: ルートの出発地を決める。初期値=現在地。
- **手段トグル**（徒歩 / 車）: ルートの移動手段。初期値=徒歩。
- トグルは同系統の丸みのあるピル型。選択中は紫（`--route`）で塗る。

**地図**
- 画面中心に**うっすらとした小さな十字レティクル**（opacity 0.4程度、pointer-events:none）。
  - これは**地図中心を示す目印（表示のみ）**。検索中心そのものではない。
  - 「中心を指定」ON中は◎ピンが中心を示すため、**レティクルは隠す**。

**右下のボタン（縦積み）**
- 「A→B ルート沿い」（青）／「◎ 中心を指定」（橙）／「● 現在地へ」（青）。
- モード中は塗りつぶし＋ラベルを「やめる」「中心を解除」に変える。

**ステータス表示（画面下部）**
- 常に**日本語で今の状況**を表示する。エラーも必ず日本語で、次に何をすればいいか書く。

**トースト（画面中央やや上）**
- 赤文字・白背景の一時メッセージ。**1.5秒で自動的に消える**。
- 用途: 出発地が「指定中心」なのに◎ピンが無い状態でルートを表示しようとしたとき。
  文言例: 「中心が未指定です。「中心を指定」でピンを置いてください」

## 5. 機能仕様

### 5.1 現在地

- 起動時に `navigator.geolocation.getCurrentPosition` で取得。
- 青い点＋精度円で表示。「現在地へ」ボタンで再取得＆移動。
- 取得失敗時は東京駅（35.681, 139.767）を初期表示し、日本語で理由を説明。
- **位置情報はhttps環境でしか動かない**点を依頼者に伝えること。

### 5.2 探す中心（重要な仕様変更あり）

- **既定（「中心を指定」OFF）: 探す中心は「現在地」。**
  - **地図を動かしても検索中心は動かない。**（当初は地図中心で検索していたが、
    依頼者から「地図を動かすと勝手に中心が変わる」と指摘があり修正した。）
- **「中心を指定」ON**: 地図中心に橙の◎ピンを置く。
  - **地図タップ**または**ピンのドラッグ**で移動できる。
  - この◎ピンが検索中心になる。
- 優先順位: `◎ピン > 現在地 > 地図中心（保険）`

### 5.3 周辺コンビニ検索

- 「このあたりを探す」で、検索中心から**スライダーの半径**内のコンビニを検索。
- 探索範囲を薄い緑の円で表示。
- **データ源の切り替え**:
  - `YAHOO_CLIENT_ID` あり → YOLP（JSONP）。`gc=0205001`（コンビニ）、`dist`はkm単位、`results=100`。
  - なし → Overpass（`shop=convenience` ＋ 主要チェーンのブランド名）。複数エンドポイントを
    順に試すフォールバック（`overpass-api.de` → `overpass.kumi.systems` → `overpass.private.coffee`）。
- **重複除去（必須）**: YOLPは同じ店を複数返すことがある。
  **「名前（空白除去して比較）が同じ」かつ「30m以内」なら同一店として1件にまとめる。**
  （座標完全一致での除去では不十分だったため、この方式に変更した経緯がある。）
- 現在地がある場合、**近い順にソート**。
- 結果は緑のピン。件数をステータスに表示。YOLP使用時は小さく「データ: Yahoo! JAPAN」と表示（帰属表示）。
- 結果の下に「**Google Mapでこの周辺も見る**」ボタン。
  → `https://www.google.com/maps/search/コンビニ/@{lat},{lng},16z` を新規タブで開く。
  （アプリ内で漏れがあった場合の逃げ道。無料・キー不要。）

### 5.4 コンビニのポップアップ

ピンをタップすると吹き出しを表示。内容:
- 店名
- 現在地からのおおよその直線距離
- **「ルートを表示する」ボタン**（紫）→ アプリ内にルート描画
- **「Google Mapでルートを開く」ボタン**（グレー）

> **重要**: 「ここへナビ」（ナビ自動開始）は**依頼者が明確に不要と指示**した。
> Google Mapは**ルートが表示されるだけ**にすること。
> → `dir_action=navigate` を**付けない**。
> `https://www.google.com/maps/dir/?api=1&origin={出発地}&destination={lat,lng}&travelmode={walking|driving}`

### 5.5 アプリ内ルート表示

- OSRM (FOSSGIS) でルートを取得し、**紫の線**で描画。目的地に紫の点。
  - 徒歩: `https://routing.openstreetmap.de/routed-foot/route/v1/foot/{lng},{lat};{lng},{lat}?overview=full&geometries=geojson`
  - 車: `https://routing.openstreetmap.de/routed-car/route/v1/driving/...`
- ステータスに「**{店名} まで {徒歩|車}ルート: 約 {距離} ・ 約 {時間}**」を表示。
- 地図左下に「× ルートを消す」ボタン。
- **出発地**は「出発トグル」に従う（現在地 or ◎ピン）。
- **手段**は「手段トグル」に従う（徒歩 or 車）。
- **ルート表示中にトグルを切り替えたら、その場で自動的にルートを引き直す**（必須）。
  - そのため「今どの店のルートを表示中か」を保持しておくこと。
- ルート計算に失敗したら、日本語で説明し「Google Mapでルートを開く」を案内する。

### 5.6 ルート表示履歴（localStorage）

- **保存タイミング: ルートを表示したときのみ**（検索時は保存しない）。
- キー: `konbini_routes_v1` / 最大30件 / 新しいものが先頭。
- 保存内容: `{ t:日時, name:店名, mode:'foot'|'car', origin:'me'|'center',
  dist:距離(m), dur:所要(秒), d:[lat,lng], geo:ルートのGeoJSON }`
- **ヘッダーの「履歴」ボタン**で下からせり出すシートを開く。
- 一覧の各行:
  - 1行目: **目的地名** ＋ 小さな**出発地バッジ**
  - 2行目: **移動手段 ・ 距離 ・ 所要時間 ・ 日時**
  - 右端に「削除」ボタン。シート上部に「すべて消去」（確認ダイアログあり）。
- **出発地バッジの色（依頼者の細かい指定あり）**:
  - 目立たせない。**同系統（寒色）でわずかに違う色**にする。
  - 現在地 → 背景 `#E8EEF6` / 文字 `#456492`（淡い青系）
  - 中心 → 背景 `#EAE7F2` / 文字 `#5a5288`（淡い藍系）
- 行をタップ → **保存済みのGeoJSONからそのルートを地図に再描画**（再計算しない）。

### 5.7 A→B ルート沿いのコンビニ検索

- 右下「A→B ルート沿い」ボタンでモードON。
- 地図タップで **A（青ピン）** → もう一度タップで **B（濃紺ピン）**。
- Bを置いた時点で自動実行:
  1. A→Bのルートを OSRM（手段トグルに従う）で取得し、**青い線**で描画。地図をフィット。
  2. **ルート沿いのコンビニを検索**:
     - **YOLP時**: ルートを等間隔にサンプリング（最大20点、間隔はスライダー値以上）し、
       各点で円検索して結果を統合。進捗を「（3/20）」のように表示。
     - **OSM時**: Overpassの `around:{半径},{lat,lng,lat,lng,...}`（ポリライン沿い検索）で1回。
  3. 重複除去して緑ピンで表示。件数をステータスに表示。
- 「探す範囲」スライダーの値を**ルートからの幅**として使う。
- 表示された店のピンからは、通常どおり個別ルート表示もできる。
- さらに地図をタップすると A から置き直し。「やめる」でモード終了。
- **中心指定モードとは排他**（片方をONにしたら他方をOFFにする）。

## 6. コード上の作法（依頼者の要望）

- **ファイル冒頭に大きなコメントブロックを置く。** 内容:
  - 現在のバージョン
  - このファイルでできること
  - 次にやること
  - **★ 再開のしかた** … 「このファイル全体をコピーしてAIに貼り、続きから追加してと伝える」
    （**依頼者は利用上限で作業が中断することがあり、このファイル自体をセーブデータとして使う**）
  - **バージョン履歴**（v0.1.0からの全履歴を残す）
  - 使用している無料サービス一覧
- **バージョン番号を3箇所に書く**: 冒頭コメント / 画面右上の表示 / JSの `APP_VERSION` 定数。
- 機能追加ごとにバージョンを上げる（機能追加=マイナー、修正=パッチ）。
- **コードのコメントはすべて日本語**。未経験者が読んで意味が分かるように書く。
- 納品物は **`index.html` 1ファイルのみ**（ZIP不要）。

## 7. 既知の限界（依頼者に必ず正直に伝えること）

- **OSMのコンビニデータは日本では大量に欠落している。** YOLPを使うこと。
- **YOLPでも取りこぼしはゼロにならない。** 「Google Mapでこの周辺も見る」で補完する。
- **OSRM/Overpassの公開サーバーは混雑時に失敗する。** リトライ or 代替導線を用意する。
- **A→B沿線検索（YOLP時）は複数回の円検索**のため、長距離だとやや時間がかかり、
  サンプル上限（20点）の関係で隙間が生じうる。幅を広げると改善する。
- **位置情報はhttps必須**（ローカルでファイルを直接開くと現在地が取れない）。
- **履歴は端末内のみ。** 他端末と共有されず、ブラウザのデータ消去で消える。

## 8. デプロイ手順（依頼者の現在の環境）

**GitHub + Vercel（同じURLで更新できる）**
1. GitHubリポジトリ（例 `konbini-app`）に `index.html` を置く。
2. Vercelでそのリポジトリを Import（Framework Preset は **Other**、ビルド設定なし）。
3. 以降、**GitHubの `index.html` を差し替えてCommitするだけで、Vercelが自動で同じURLを更新**する。

> Netlifyの Drag&Drop も使えるが、依頼者は無料枠を使い切ったためVercelに移行済み。
> Vercel Drop（vercel.com/drop）は**毎回新しいプロジェクトを作る**ので更新には使えない点に注意。
